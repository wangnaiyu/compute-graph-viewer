Scope · 基本对象

card、device、rank、worker、stage、expert 的层级关系

若产品界面仅使用“卡”作为统一描述，硬件位置、运行时身份、层段分工与专家放置容易被混合表达。本节先建立对象层级，作为后续 placement 建模的基础。

对象层级图 · 物理设备、运行时身份与并行坐标
读图顺序：左侧表示物理设备，中间表示运行时身份，右侧表示混合并行配置下的多维放置标签；这些标签是否全部出现，取决于实际训练配置。
Ascend / MindSpore 依据与边界：官方文档支持多种并行模式组合，配置项覆盖 data_parallel、model_parallel、context_parallel、pipeline_stage 和 MoE 专家相关字段。右侧“并行坐标层”是产品抽象；实际页面应按配置有则展示、无则隐藏。参考：分布式并行训练、配置文件说明。
对象更容易理解的说法在页面里该怎么用
card / device 一块物理硬件，或者承载它的主机。 表示“这份任务跑在哪台机器 / 哪张卡上”。
rank / worker 运行时里的进程身份，负责参与训练的那个执行体。 表示“当前执行单元是谁”，不是“它拥有哪一层”。
stage 流水线并行中的一个连续层段。 只在 PP 语义下直接对应层范围，例如 layers 36-42。
expert MoE 里的专家子模块。 表示 token 被路由到哪些专家，以及这些专家落在哪些 rank 上。

Reference · 术语坐标系

读图前先对齐词汇：哪些词在切“数据”，哪些在切“模型”

前面每张图都假设你已经分清 sample / token / batch / step / stage，以及 block / layer / hidden state 的区别。这些词常被混在一起，但它们各自回答的是不同维度的问题。本节作为全篇的术语落点。

概念准确定义属于哪个维度图里怎么写
sample一条训练数据，对语言模型通常是一段 token 序列。数据维度sample 0-511
tokentokenizer 切出的最小训练单位（字 / 词片段 / 符号）。数据维度token 2048-4095
tensor承载数据 / 权重 / 激活 / 梯度的多维数组；样本编码后变成 tensor，但 tensor ≠ 样本。计算表示hidden_states tensor
batch一个 step 一次送进模型的一组样本。数据维度one step · 32 samples
mini-batch shardDP / CP 下，一个 rank 实际处理的 batch 子集。数据维度rank_1 · samples 512-1023
step一次训练迭代：取 batch → forward → loss → backward → 通信 → optimizer。时间维度step 1997
stagePP 把整网层序列切成的一个流水线阶段。模型切分PP5 · Blocks 20-23
一句话区分：sample / batch / mini-batch shard 回答“喂了哪些数据”，tensor 回答“数据在机器里是什么数组形态”，stage 回答“整网哪一段层在这个阶段”，step 回答“跑到第几次迭代”。所以 DP 图里“不同 rank 吃不同数据”说的是 mini-batch shard 不同，不是模型结构不同。
词准确含义建议用法
block / decoder blockTransformer 里重复堆叠的完整计算单元（Attention、FFN/MoE、Norm、Residual）。描述 PP 切分层段优先用，例如 Blocks 20-23。
layer语境依赖强：可指整个 block，也可指更小的 linear / attention / norm 子层。不裸写“layer”，要说明是 decoder layer 还是 linear / attention sublayer。
num_hidden_layersHugging Face 配置字段，表示模型里的 block 数量。num_hidden_layers=61 即 61 个 decoder blocks。
隐藏层 / hidden layer中文里易被误用来指 hidden state 或 num_hidden_layers。避免做产品标签；改写成“隐藏状态 hidden state”或“层数量字段 num_hidden_layers”。
hidden state每个 token 在模型内部的中间向量，形状 [batch, seq_len, hidden_size]，是 runtime tensor 不是模块。描述激活、attention 输入输出、层间传递数据时用。
hidden_sizehidden state 每个 token 向量的宽度（最后一维）。解释张量形状 / 模型宽度时用，例如 Pangu hidden_size=7680。
因此 Blocks 20-23 不能翻译成“隐藏状态 20-23”：它指 PP5 这个 stage 覆盖的第 20-23 个 decoder block；这些 block 接收上一层的 hidden state，执行 Attention 和 FFN/MoE，再输出新的 hidden state。来源：knowledge.md §1.2、§10.5.5。
